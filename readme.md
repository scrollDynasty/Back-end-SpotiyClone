# Spotify Clone Back-end

Этот проект представляет собой бэкенд для клона Spotify, разработанный с использованием современных технологий и инструментов. Основная цель проекта — предоставить API для управления пользователями, аутентификации и загрузки файлов (например, музыки).

## Основные функции

- **Аутентификация пользователей**: Регистрация и вход с использованием JWT (JSON Web Tokens) для безопасной аутентификации.
- **Загрузка файлов**: Возможность загрузки музыкальных файлов на сервер с использованием `multer`.
- **Работа с базой данных**: Использование MongoDB для хранения данных пользователей и другой информации.
- **API документация**: Интерактивная документация API с использованием Swagger/OpenAPI.

## Технологии

- **Node.js**: Среда выполнения JavaScript на стороне сервера.
- **Express.js**: Фреймворк для создания веб-приложений и API.
- **MongoDB**: NoSQL база данных для хранения данных.
- **Mongoose**: ODM (Object Data Modeling) библиотека для работы с MongoDB.
- **JWT (JSON Web Tokens)**: Используется для аутентификации и авторизации пользователей.
- **Bcrypt**: Библиотека для хеширования паролей.
- **Multer**: Middleware для обработки загрузки файлов.
- **CORS**: Middleware для обработки кросс-доменных запросов.
- **Swagger/OpenAPI**: Инструмент для документирования и тестирования API.
- **dotenv**: Модуль для работы с переменными окружения.

## Установка и запуск

1. **Клонирование репозитория**:
   ```bash
   git clone https://github.com/scrollDynasty/spotify-clone-backend.git
   cd spotify-clone-backend
   ```

2. **Установка зависимостей**:
   ```bash
   npm install
   ```

3. **Настройка переменных окружения**:
   - Создайте файл `.env` в корне проекта со следующим содержимым:
     ```
     MONGODB_URI=MONGODB_URI
     JWT_SECRET=your_secure_jwt_secret_key
     PORT=4000
     ```
   - Замените `your_secure_jwt_secret_key` на безопасный секретный ключ для JWT.
   - При необходимости измените URI для MongoDB и порт.

4. **Запуск сервера**:
   ```bash
   npm run dev
   ```
   или
   ```bash
   npm run start:dev
   ```
   Сервер будет запущен на порту, указанном в `.env` файле (по умолчанию 4000).
   
5. **Доступ к документации API**:
   После запуска сервера документация API будет доступна по адресу:
   ```
   http://localhost:4000/api-docs
   ```

## API Endpoints

### Аутентификация

- **POST /auth/register**: Регистрация нового пользователя.
  - Тело запроса:
    ```json
    {
      "email": "user@example.com",
      "fullName": "John Doe",
      "password": "password123",
      "avatarUrl": "http://example.com/avatar.jpg" // Необязательное поле
    }
    ```

- **POST /auth/login**: Вход пользователя.
  - Тело запроса:
    ```json
    {
      "email": "user@example.com",
      "password": "password123"
    }
    ```

### Загрузка файлов

- **POST /upload**: Загрузка музыкального файла (требуется авторизация).
  - Заголовок: `Authorization: Bearer ваш_токен`
  - Формат запроса: `multipart/form-data`
  - Поле: `music` (файл)

## Использование Swagger

Swagger предоставляет интерактивную документацию API, которая позволяет легко тестировать все эндпоинты.

1. **Доступ к Swagger UI**:
   Откройте в браузере адрес: `http://localhost:4000/api-docs`

2. **Авторизация**:
   - Получите токен, выполнив запрос к `/auth/register` или `/auth/login`
   - Нажмите кнопку "Authorize" в верхней части страницы
   - Вставьте полученный токен в поле "Value" (без префикса "Bearer")
   - Нажмите "Authorize", затем "Close"

3. **Тестирование эндпоинтов**:
   - Выберите нужный эндпоинт
   - Нажмите "Try it out"
   - Заполните необходимые параметры
   - Нажмите "Execute"
   - Просмотрите результат запроса

## Структура проекта

- **index.js**: Основной файл сервера, содержащий настройки и маршруты.
- **package.json**: Файл с зависимостями и скриптами для запуска проекта.
- **control/control.js**: Контроллеры для обработки запросов на регистрацию и вход.
- **models/user.js**: Модель пользователя для MongoDB.
- **validations/auth.js**: Валидаторы для проверки данных при регистрации и входе.
- **utils/checkAuth.js**: Middleware для проверки авторизации.
- **swagger.js**: Конфигурация Swagger для документирования API.
- **.env**: Файл с переменными окружения (не включен в репозиторий).
